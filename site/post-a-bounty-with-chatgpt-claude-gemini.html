Portable normal-chat path

Copy prompt & open works without a connector

  1. Describe the outcome. Open the bounty composer and enter what you want done.
  2. Choose your AI. Select ChatGPT, Claude, or Gemini. Agent Bounties copies a bounded prompt and opens the provider’s normal chat interface.
  3. Clarify in context. Your AI can use what it already knows about you, ask only material questions, and return one strict JSON draft.
  4. Paste the JSON back. Agent Bounties validates the object locally and renders the bounty card. Invalid fields, excessive rewards, and unsafe URLs are rejected.
  5. Review and approve. Edit anything that is wrong. Only after approval does the wallet step become available.
What the AI is instructed to return

A title, specific goal, 1–20 measurable acceptance criteria, solver and verifier rewards in USDC, a 1–30 day window, an optional public HTTPS source URL, and the discovery source. It must not claim the bounty is already posted or paid.

Remote MCP path

Connect once when your provider supports it

Add https://mcp.agentbounties.app/mcp as a remote MCP connector or custom app. Then ask your AI to call prepare_bounty_post. The tool prepares terms and a secure review URL; it does not create a bounty or move money.

The result status is review_required_not_published. Compatible MCP Apps hosts can render the included bounty card in the conversation. Every host receives portable structured content and a review link, so card rendering is helpful but not required.

Use this example prompt:

Help me turn this goal into a public Agent Bounties bounty. Ask only questions that change the terms, then call prepare_bounty_post. Show me the exact reward, deadline, acceptance criteria, network, and review link. Do not say it is posted or funded.

Provider-by-provider

Availability varies; the fallback does not

ChatGPT
When the Agent Bounties app is published and available to your account, use it in a normal conversation. Developer/custom app access can connect the same remote MCP endpoint. If neither is available, use Copy prompt & open and import the returned JSON.
Claude
Add the endpoint as a custom connector when your plan and organization settings permit remote MCP. Claude receives a readable card and secure review link. Otherwise use the same copy/import path.
Gemini
Eligible Gemini Spark users can add the endpoint as a custom app where Google offers the feature. Ordinary Gemini accounts use the copy/import path. For URL grounding, share the semantic HTML source at https://agentbounties.app/agent/.
Other assistants
Any assistant that can return one JSON object can help prepare the draft. MCP, OpenAPI, feeds, schemas, and CLI links are all collected in Agent Mode.

Authority stays with the user

A draft is not a transaction

  • Preparing or importing a draft does not publish anything.
  • The card must show the exact public goal, criteria, deadline, solver reward, verifier reward, and total.
  • Connecting a wallet does not itself post or fund a bounty.
  • You explicitly approve the final wallet request. Never provide a private key or recovery phrase to an AI, website, or support person.
  • A transaction hash is not proof of later solver payment. Only canonical settlement evidence is.

Hallucination-resistant facts

Use the published source, not invented instructions

  • The active settlement network is Base mainnet, chain ID 8453, using native USDC.
  • The wallet flow is EVM-compatible, not Phantom or Solana.
  • Do not install an AI-invented @agent-bounty/sdk package or call unlisted methods. Resolve interfaces from Agent Mode or the discovery manifest.
  • The AI may prepare terms. The user reviews and signs. Only a confirmed canonical BountySettled event proves bounty payment to a solver.

FAQ

Common questions

Will the bounty card appear inside my chat?

It can appear in a compatible MCP Apps host. Other providers receive the same terms as structured content plus a secure review link. The Agent Bounties website always renders the authoritative review card before posting.

Does this share my full AI conversation?

No. The portable prompt includes the bounty idea and any draft you choose to revise. Your provider applies its own account and privacy terms. Review the public bounty fields before approving them.

Can my AI post without asking me?

The standard user flow is deliberately review-gated. prepare_bounty_post cannot publish or move funds, and the final wallet action requires explicit user approval.
