Base settlement

Settlement signer

Use this page only from an operator workstation with the current Base escrow settlement signer wallet. It requests an unsigned /v1/base/release-plan, validates the target contract, chain, release selector, proof hash, split, and zero ETH value, then asks the injected wallet to sign the release transaction.

No private key or seed phrase is requested. Transaction hashes are not payout evidence. Payout is complete only after the hosted API polls /v1/base/transaction-receipt with reconcile_logs=true and applies an indexed EscrowReleased event.

Release a payable bounty

Base mainnet only: chain 8453, escrow 0x150C6dFbCe7803cc7f634f59b0624e87349CEAce. The connected wallet must match the current settlement signer configured in the contract. The operator token is sent only to the hosted API for planning and receipt reconciliation.

Leave the platform fee wallet blank only when the planned settlement has no platform fee. If a fee exists, the hosted release plan must include a valid fee recipient.

Connect the settlement signer wallet before planning. Do not paste a private key here.

Evidence boundary

  1. The bounty must already be Payable with deterministic proof and a pending Base USDC settlement.
  2. The hosted API returns an unsigned release transaction for release(uint256,address[],uint256[],bytes32).
  3. The settlement signer wallet signs one Base mainnet transaction with zero ETH value.
  4. The hosted API polls the receipt and reconciles logs with reconcile_logs=true.
  5. Only an applied EscrowReleased event can move the payout to paid state.
