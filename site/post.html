Checking protocol

1. Define the goal and task

The cloud draft is advisory. Review the task objective and every acceptance criterion.

The objective describes the result. The acceptance criteria are the checks that prove the result was achieved.

2. Set the reward

The verifier reward also sets the refundable solver bond.

Crowdfunding fallback: The creator still signs contract creation and pays gas. The task becomes claimable only after the full reward target is funded.

Change the 7-day funding, 24-hour work, and 2-hour verification deadlines

3. Choose how completion is verified

This choice controls reward release. The bounty pays only when the committed verification policy accepts the submitted proof.

The demo checker proves only that a small computation was completed. It does not check whether your requested task objective or acceptance criteria were satisfied.
